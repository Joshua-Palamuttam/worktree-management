### 6. Create `scripts/wt-sync-permissions.sh` + `bin/wt-sync-permissions.cmd` â€” interactive permission sync

New command that scans worktrees and lets you promote permissions to global settings:

1. Scans all `.claude/settings.local.json` files across worktrees in the current repo (or all repos with `--all`)
2. Collects all unique `permissions.allow` entries NOT already in `~/.claude/settings.local.json`
3. Presents each new permission interactively â€” user picks `y/n/a(ll)/q(uit)` for each
4. Merges selected permissions into the global settings file

Uses `node -e` for reliable JSON read/write (available on all dev machines).

Usage:
```
wt-sync-permissions          # Scan current repo's worktrees
wt-sync-permissions --all    # Scan all repos under WORKTREE_ROOT
```
